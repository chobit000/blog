hexo从零到部署（二）

  • hexo


上篇搭建了hexo的环境，这篇谈一下如何部署到HITGUB

4. 配置Github

Create a new repository 并命名为blog, 最终你的仓库路径应该是［https://github.com/你的帐号/blog]

https://github.com/chobit000/blog

5. 配置gh-pages

在仓库的Settings中, 有一栏标签为GitHub Pages, 用来给给仓库建立一个主页, 可以使用GitHub上提供的模板, 也可以使用自定义主页.
只要知道有这个功能就行, 不用配置这里, 到时候强制建立gh-pages分支即可.
自定义主页说明帮助页:
https://help.github.com/articles/creating-project-pages-manually/

6. 根目录_config.yml

URL

url: http://chobit000.github.io/blog/
root: /blog/
permalink: :year-:month-:day-:desc/
permalink_defaults:

要配置的是网站子目录, 所以url和root都要加上blog, 可以参考这里:
https://hexo.io/zh-cn/docs/configuration.html
有个细节要注意下, permalink的值是:year-:month-:day-:desc/这个格式的, 前面没有/

Writing

highlight:
  enable: true
  line_number:
  auto_detect: true
  tab_replace: "    "

Deployment

deploy:
  type: git
  repo: https://github.com/chobit000/blog
  branch: gh-pages

Theme

theme: yilia

主题的话很多用哪个看个人喜好，我用的yilia
Themes: http://hexo.io/themes/
Plugins: http://hexo.io/plugins/

7.主题_config.yml

Menu

menu:
  主页: /blog
  所有文章: /archives
  随笔: /tags

配置路径

SubNav

subnav:
  github: "https://github.com/chobit000/"
  weibo: "http://weibo.com/chobit000/"
  mail: "http://chobit000@qq.com"

几个链接的配置

你的头像url

avatar: ./images/face.gif

头像的地址

多说

duoshuo: chobit000

如果想引入评论系统可以注册多说，在这里填写多说的帐号即可。

8.部署

安装git插件

$ npm install hexo-deployer-git --save

生成和部署

$ hexo generate 
$ hexo deploy

如果部署过必须先执行clear,用来清除历史记录

$ hexo clear

调试可以起一个本地服务, 在浏览器中访问http://localhost:4000/blog/ 

$ hexo server

欢迎访问我的Blog,ByeBye
http://chobit000.github.io/blog/
